After several years on DDS-heavy stacks at Thales, one pattern became obvious: the moment you let an LLM agent touch the control bus — even with good intentions — you have a liability conversation you cannot win. The safety review alone kills most proposals. The default open-source answer today (rosbridge-based MCP servers with a full write path) is exactly that shape.

TopicForge takes a different position. It is read-only by architecture, not by configuration. There is no write path to misconfigure, no permission flag you could accidentally flip, no command path back to the robot. The MCP client can see your ROS2 graph — topics, message types, bag recordings, live samples with real publish-time timestamps as of v0.1.2 — but it cannot touch it. That is the sentence that ends the security review.

This is MCP 01 of a small pack. The next piece addresses DDS observability below and beyond the ROS layer — same safety-first stance, broader middleware reach.
